# 커리큘럼 및 학습 설계 전략 (Curriculum & Learning Strategy)

본 프로젝트는 단순한 지식 전달을 넘어, 웹 엔지니어가 복잡한 시스템의 동작 원리를 꿰뚫어 볼 수 있는 **'시스템 사고(System Thinking)'** 능력을 배양하는 것을 최우선 목표로 합니다.

## 1. 학습 철학: 시스템 사고 (System Thinking)
웹 애플리케이션은 브라우저, 네트워크, 서버, 데이터베이스 등 수많은 레이어가 유기적으로 연결된 거대한 시스템입니다.
- **왜(Why) 배우는가?**: 자료구조와 알고리즘은 코딩 테스트용 도구가 아니라, 성능 최적화와 안정적인 아키텍처 설계를 위한 필수 언어입니다.
- **어떻게(How) 배우는가?**: 추상적인 개념을 시각적 시뮬레이션으로 형상화하고, 이를 실제 프론트엔드/백엔드 실무 사례와 즉각적으로 매핑합니다.

## 2. 2-트랙 전략 (2-Track Strategy)

### 🟢 트랙 1. Base Systems (기초 체력)
웹 개발에 필수적인 CS 기초 개념을 '이해' 중심으로 다룹니다.
- **핵심 목표**: 데이터의 흐름과 컴퓨터 메모리의 작동 원리를 직관적으로 파악합니다.
- **주요 모듈**: 시간복잡도, 배열, 스택, 큐, 해시테이블, 트리.

### 🔵 트랙 2. Applied Web Systems (실무 응용)
기초 개념이 실제 웹 기술 명세(Spec)와 프레임워크 내부에서 어떻게 작동하는지 심층 분석합니다.
- **핵심 목표**: "코드 한 줄이 시스템 전체에 어떤 영향을 미치는가?"에 답할 수 있게 합니다.
- **주요 모듈**: 이벤트 루프(JS Runtime), 브라우저 렌더링 파이프라인, 네트워크 요청 생애주기, DB 성능 최적화.

## 3. 상세 커리큘럼 로드맵

### Level 1. Foundations (기초)
| 모듈 | 학습 포인트 | 실무 매핑 사례 |
| :--- | :--- | :--- |
| **시간복잡도** | Big-O 감각과 성능 병목 지점 탐색 | React 리스트 렌더링 비용 계산 |
| **배열 & 리스트** | 연속 메모리 구조와 접근 효율성 | 불변 데이터 흐름과 주소값 비교(Memoization) |
| **스택 & 큐** | LIFO/FIFO 원리와 데이터 처리 순서 | Call Stack 예외 처리, 비동기 작업 큐 |
| **해시테이블** | Key-Value 저장 및 탐색의 원리 | JavaScript Map/Object, 캐싱 전략 |
| **트리 구조** | 계층형 데이터 최적 탐색 | DOM / Virtual DOM Diffing 알고리즘 |

### Level 2. Web Runtime & Network (심화)
| 모듈 | 학습 포인트 | 실무 매핑 사례 |
| :--- | :--- | :--- |
| **Runtime** | 비동기 실행 메커니즘 (Event Loop) | Microtask vs Macrotask 우선순위 |
| **Rendering** | Critical Rendering Path (CRP) 분석 | Layout Thrashing 방지 및 Paint 최적화 |
| **Network** | TCP/TLS 핸드셰이크와 요청 생애주기 | HTTP/2 Multiplexing, 프리페칭(Prefetching) 전략 |
| **Persistence** | B-Tree와 데이터 인덱싱 원리 | DB Query 성능 튜닝, 페이지네이션 설계 |

### Level 3. System Architecture (확장)
- **E2E 흐름 시뮬레이션**: 유저 클릭 -> API 요청 -> 서버 처리 -> DB 조회 -> 렌더링의 전 과정 시각화.
- **전략적 캐싱**: 브라우저, CDN, 서버, DB에 이르는 다층 캐시 설계 전략.

## 4. 교수법 및 학습 프로세스
모든 레슨은 다음 3단계 과정을 따릅니다.

1.  **Concept Exploration (개념 탐색)**: 친절한 한글 설명과 함께 기술적 모티브를 제안합니다.
2.  **Interactive Simulation (동적 시뮬레이션)**: 브라우저 상에서 데이터가 변하는 모습을 직접 단계별로 실행하며 관찰합니다.
3.  **Practical Mapping (실무 매칭)**: 학습한 개념이 실제 웹 개발 프로젝트에서 어떤 코드로 나타나고, 어떤 성능 차이를 만드는지 정리합니다.
